# 文献译读 · 学生版 PRD

## 1. 产品定位

面向**国内大学生/研究生**，以**英文学术文献**为主场景：读论文、做作业、整理阅读笔记。核心差异点：**多格式导入 + 全文可搜 + 左右同字段联动**（框选左侧，右侧显示同一字段译文）。

## 2. 目标用户与场景

| 场景 | 行为 | 产品能力 |
|------|------|----------|
| 读 PDF 论文 | 划选一段看译文 | 字段联动 + 段落级分段 |
| 读 Word 讲义 | 表格、标题结构清晰 | DOCX 解析为字段（含表格单元格） |
| 复习多篇文献 | 记得某个术语在哪篇 | 文献库级全文搜索 |
| 写读书笔记 | 微调译文 | 右侧译文可编辑、本地保存 |

## 3. 核心概念：「字段」

将文档拆成带稳定 ID 的**字段（Segment）**，左右栏按同一 `segmentId` 对齐：

- **段落** `paragraph`：论文正文
- **标题** `heading`：章节标题
- **表格单元格** `table-cell`：带 `tableId / row / col`
- **列表项** `list-item`、**引用** `quote` 等

用户**点击或框选**左侧某字段 → 右侧滚动并高亮**相同字段**的译文。

## 4. 导入格式（MVP 已实现 / 规划）

| 格式 | MVP | 说明 |
|------|-----|------|
| PDF | ✅ | pdf.js 按页提取文本（需可选中文本层） |
| DOCX | ✅ | mammoth → HTML → 字段拆分 |
| TXT / MD | ✅ | 空行分段 |
| HTML | ✅ | DOM 解析字段 |
| RTF | ✅ | 去控制符后按 TXT 处理 |
| EPUB | 🔜 | 下一版本 |
| 扫描版 PDF | 🔜 | 需 OCR 后端 |

## 5. 搜索

- **当前文献**：Fuse.js 模糊搜原文 + 译文
- **全部文献**：跨文献库搜索，点击结果跳转并联动字段
- **上一处 / 下一处**：在命中结果间循环

## 6. 翻译引擎（国内）

| 阶段 | 方案 |
|------|------|
| MVP | 占位译文 + 手动编辑（已预留 API） |
| V1 | Node 代理 **百度翻译 API**（学术领域可选） |
| V1.1 | 可选 **有道** 作对照引擎 |

数据：**文献解析与存储默认仅浏览器 localStorage**，不上传原文（除用户主动调用翻译 API 的文本片段）。

## 7. 信息架构

```
顶栏：导入 | 翻译全文 | 翻译当前字段
侧栏：文献库列表
主区：搜索条
      ├─ 左：原文字段列表
      └─ 右：译文字段列表（同 ID）
```

## 8. 非功能需求

- 国内访问：静态前端可部署在阿里云/腾讯云 CDN
- 性能：单篇 &lt; 500 字段流畅滚动；PDF 大文件提示分页优化
- 无障碍：字段支持键盘 Enter 聚焦

## 9. 版本路线

- **MVP（当前）**：导入、字段联动、搜索、本地文献库、译文编辑
- **V1**：百度 API、导出 bilingual DOCX/MD
- **V2**：EPUB、OCR、术语本、划词插件
